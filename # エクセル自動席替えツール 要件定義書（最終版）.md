# エクセル自動席替えツール 要件定義書（最終版）

---

## 1. 概要

社員の席替えを自動で行うExcel VBAツールを作成する。
社員情報シート上のボタン押下により、席順シートを新規生成し、席替え結果を出力するとともに社員情報シートの現在のテーブル、前回のテーブル、前々回のテーブルのデータを更新する。

---

## 2. シート構成

### 2.1 社員情報シート

#### シート名

`社員情報`

#### データ構造

* 横にカラム（項目）
* 縦にレコード（社員情報）
* データベース形式で管理する

---

### ■ レイアウト仕様（重要）

#### 1～4行目：UIエリア

| セル | 内容                |
| -- | ----------------- |
| A1 | タイトル（例：社員情報）      |
| B4 | 席替えボタン            |
| D4 | 経験者分散チェックボックス     |
| E4 | 前回テーブル回避チェックボックス  |
| F4 | 前々回テーブル回避チェックボックス |

---

#### 6行目：ヘッダー

| 列  | 項目       |
| -- | -------- |
| A列 | 社員番号     |
| B列 | フリガナ     |
| C列 | 名前       |
| D列 | 経験有無     |
| E列 | 現在のテーブル  |
| F列 | 前回のテーブル  |
| G列 | 前々回のテーブル |
| H列 | 性別       |
| I列 | 研修グループ   |

---

#### 7行目以降：データ

```text
A7: 社員番号
B7: フリガナ
C7: 名前
D7: 有 / 無
E7: 現在テーブル
F7: 前回テーブル
G7: 前々回テーブル

I7: 研修グループ
```

---

### ■ 入力ルール

* 経験有無：`有` または `無`
* テーブル番号：1〜14
* 初回はテーブル情報は空欄可

---


### 2.2 席順シート

#### シート名

`席順_YYYYMMDD_フリガナ`

#### 日付仕様

* `YYYYMMDD`は席替え実行日とする
* 席替えボタン押下時のシステム日付を使用して生成する

#### フリガナ部分

* `フリガナ` はリテラル文字列として固定出力する（例：`席順_20260404_フリガナ`）

## 3. UI仕様（社員情報シート）

### 3.1 席替えボタン

* 配置場所：社員情報テーブルの上部
* 機能：

  * 席替え処理の実行
  * 席順シートの生成
  * 履歴更新

---

### 3.2 制約チェックボックス

* 配置場所：席替えボタンの右側

#### 項目一覧

| 項目        | 内容              |
| --------- | --------------- |
| 経験者分散     | 経験者をエリア分散する     |
| 前回テーブル回避  | 前回と同じテーブルを禁止    |
| 前々回テーブル回避 | 前々回テーブルを可能な限り回避 |

---

### 3.3 チェックボックス仕様

| 項目        | リンクセル | ON値  | OFF値  |
| --------- | ----- | ---- | ----- |
| 経験者分散     | D4    | TRUE | FALSE |
| 前回テーブル回避  | E4    | TRUE | FALSE |
| 前々回テーブル回避 | F4    | TRUE | FALSE |

---

### 3.4 初期状態

| 項目        | 初期値  |
| --------- | ---- |
| 経験者分散     | TRUE |
| 前回テーブル回避  | TRUE |
| 前々回テーブル回避 | TRUE |

---

## 4. 座席構成

* 最大人数：42人
* テーブル数：14（左右2列 × 7行）
* 1テーブル：3人

---

### 4.1 配置位置

| 行 | 左テーブル   | テーブル番号 | 右テーブル   | テーブル番号 |
| - | ------- | ------ | ------- | ------ |
| 1 | C4:E4   | 1      | G4:I4   | 2      |
| 2 | C5:E5   | 3      | G5:I5   | 4      |
| 3 | C6:E6   | 5      | G6:I6   | 6      |
| 4 | C7:E7   | 7      | G7:I7   | 8      |
| 5 | C8:E8   | 9      | G8:I8   | 10     |
| 6 | C9:E9   | 11     | G9:I9   | 12     |
| 7 | C10:E10 | 13     | G10:I10 | 14     |

※ テーブル番号は左→右、上→下の順に1〜14を割り当てる

---

### 4.2 人数不足時の対応

- 使用テーブル数 = 切り上げ（人数 ÷ 3）
- テーブルは1から順に使用する
- 未使用テーブルは後方（テーブル番号が大きい順）とする
- 未使用テーブルには一切配置しない

---

## 5. セル出力形式

1セルに以下を改行して記載する：

```text
社員番号
フリガナ
名前
```

---

## 6. 席替えロジック

### 6.1 優先順位

1. 経験者の分散（最優先）
2. 前回テーブル回避
3. 前々回テーブル回避（違反可）

※チェックボックスON時のみ適用

---

### 6.2 エリア分割

全体を以下の4エリアに分割する：

* 左前（1〜3行目 左列）
* 右前（1〜3行目 右列）
* 左後（4〜7行目 左列）
* 右後（4〜7行目 右列）

#### エリア定義（テーブル番号）
- 左前：1,3,5
- 右前：2,4,6
- 左後：7,9,11,13
- 右後：8,10,12,14
---

### 6.3 経験者分散ルール

* 経験有の社員を4エリアに均等配置
* 人数が均等にならない場合：

  * 可能な限り均等（例：3,3,2,2）

---

### 6.4 テーブル制約

* 前回のテーブル：一致禁止（ON時）
* 前々回のテーブル：可能な限り回避（ON時）

---

### 6.5 配置手順

1. 社員情報を取得
2. 制約設定を読み込む
3. 経験有 / 無で分割
4. 経験者を優先配置（ON時）
5. 残りをランダム配置
6. 制約チェック
7. 違反時は再配置（リトライ）

---

## 7. VBA処理仕様

### 7.1 実行トリガー

* 社員情報シート上のボタン押下

---

### 7.2 設定読み込み

```vba
useExperienceRule = Range("D4").Value
avoidLastTable = Range("E4").Value
avoidPrev2Table = Range("F4").Value
```
---

### 7.3 シート生成

* 新規作成：
  `席順_YYYYMMDD_フリガナ`

* 日付：

  * VBAの `Date` を使用

* 同名シートが存在する場合（同日再実行）：

  * 警告ダイアログを表示する
  * OKを押した場合：既存シートを上書きして処理を続行する
  * キャンセルを押した場合：処理を中断する

```vba
If SheetExists(sheetName) Then
    If MsgBox("同日のシートが既に存在します。上書きしますか？", vbOKCancel) = vbCancel Then Exit Sub
    Application.DisplayAlerts = False
    Worksheets(sheetName).Delete
    Application.DisplayAlerts = True
End If
```

---

### 7.4 席順出力

* 指定セルに社員情報内の「社員番号」「フリガナ」「名前」を改行付きで出力

---

### 7.5 履歴更新

全社員について以下を更新：

```text
前々回 ← 前回
前回 ← 現在
現在 ← 今回のテーブル番号
```

---
### 7.6 内部データ構造

社員情報は以下の配列で扱う：

emp(1): 社員番号  
emp(2): フリガナ  
emp(3): 名前  
emp(4): 経験有無  
emp(5): 現在テーブル  
emp(6): 前回テーブル  
emp(7): 前々回テーブル  
emp(8): 性別  
emp(9): 研修グループ  
emp(10): 今回テーブル（処理中に付与）

---
## 8. エラーハンドリング

* 人数0人：処理中断
* 配置失敗：

  * 「失敗」の定義：前回テーブル一致禁止の制約を満たせない場合
  * 前々回テーブル回避は制約違反があっても許容する（ただしなるべく回避する）
  * 最大リトライ回数（例：100回）まで試行
  * 失敗時はエラーメッセージ表示

---

## 9. 拡張性

* 性別バランス調整
* チーム分散
* 固定席設定
* 再抽選機能
